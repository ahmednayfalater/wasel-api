⚡ وصل — Wasel API

نظام إدارة المولدات الكهربائية — Laravel Backend

Base URL https://wasel-api-production-0719.up.railway.app/api
📌 كل الـ requests المحمية تحتاج Header:
Authorization: Bearer {token} — تحصل عليه بعد تسجيل الدخول
Accept: application/json — مطلوب في كل request
1. المصادقة — Auth
POST /register/customer تسجيل مواطن عام
Body (JSON)
{
  "first_name": "محمد",
  "last_name": "أبو علي",
  "email": "customer@test.com",
  "password": "password123",
  "password_confirmation": "password123",
  "phone": "0599111111"
}
POST /register/provider تسجيل صاحب مولد عام
Body (form-data)
first_name        = خالد
last_name         = النجار
email             = provider@test.com
password          = password123
password_confirmation = password123
phone             = 0599222222
company_name      = مولدات النجار
price_KW          = 5.5
terms_subscr      = يتم الدفع شهرياً
proofs[]          = {صورة إثبات}
POST /login تسجيل دخول — يرجع token عام
Body (JSON)
{
  "email": "customer@test.com",
  "password": "password123"
}
POST /logout تسجيل خروج مسجّل دخول
لا يحتاج Body
GET /user بيانات المستخدم الحالي مسجّل دخول
لا يحتاج Body
2. المولدات — Generators
GET /generators كل المولدات النشطة مسجّل دخول
Query Params اختيارية
?area_id=1&status=active
GET /generators/{id} تفاصيل مولد مسجّل دخول
لا يحتاج Body
GET /generators/search?q= بحث عن مولد مسجّل دخول
مثال
GET /generators/search?q=كاتربيلر
POST /generators إضافة مولد Provider
Body (JSON)
{
  "type": "كاتربيلر 2022",
  "status": "active",
  "gps": "31.5017,34.4668",
  "powerKW": 150
}
PUT /generators/{id} تعديل مولد Provider
Body (JSON)
{
  "status": "maintenance"
}
DELETE /generators/{id} حذف مولد Provider
لا يحتاج Body
3. الاشتراكات — Subscriptions
POST /subscriptions طلب اشتراك Customer
Body (JSON)
{ "generator_id": 1 }
GET /subscriptions/my اشتراكاتي Customer
لا يحتاج Body
DELETE /subscriptions/{id} إلغاء اشتراك Customer
لا يحتاج Body
PUT /subscriptions/{id}/approve قبول اشتراك Provider
لا يحتاج Body
PUT /subscriptions/{id}/reject رفض اشتراك Provider
لا يحتاج Body
4. الفواتير — Invoices
POST /invoices إصدار فاتورة Provider
Body (JSON)
{
  "subscription_id": 1,
  "previous_reading": 100,
  "current_reading": 250,
  "due_date": "2026-07-01"
}
GET /invoices/my فواتيري Customer
لا يحتاج Body
GET /invoices/{id} تفاصيل فاتورة Customer
لا يحتاج Body
5. الدفعات — Payments
POST /payments رفع إثبات دفع Customer
Body (form-data)
invoice_id     = 1
amount         = 825
receipt_image  = {صورة الإيصال}
GET /payments/my سجل دفعاتي Customer
لا يحتاج Body
PUT /payments/{id}/review مراجعة دفعة Provider
Body (JSON)
{ "status": "approved" }
{ "status": "rejected" }
6. التقييمات والشكاوى
POST /reviews إضافة تقييم Customer
Body (JSON)
{
  "target_type": "provider",
  "target_id": 1,
  "rate": 5,
  "comment": "خدمة ممتازة"
}
POST /complaints تقديم شكوى Customer
Body (JSON)
{
  "title": "انقطاع متكرر",
  "description": "المولد ينقطع كل ساعة",
  "target_type": "provider",
  "target_id": 1
}
7. الإشعارات — Notifications
GET /notifications/my إشعاراتي Customer
لا يحتاج Body
POST /notifications/{id}/read تحديد كمقروء Customer
لا يحتاج Body
POST /notifications/send إرسال إشعار للمشتركين Provider
Body (JSON)
{
  "message": "سيتم إيقاف المولد للصيانة غداً",
  "type": "generator_status"
}
8. الإدارة — Admin
GET /admin/users كل المستخدمين Admin
لا يحتاج Body
GET /admin/providers كل الـ providers Admin
لا يحتاج Body
PUT /admin/providers/{id}/approve قبول provider Admin
لا يحتاج Body
PUT /admin/providers/{id}/suspend تعليق provider Admin
لا يحتاج Body
GET /admin/complaints كل الشكاوى Admin
لا يحتاج Body
GET /admin/reports إحصائيات النظام Admin
لا يحتاج Body
